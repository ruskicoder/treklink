## Open Questions

> [!IMPORTANT]
> These affect final BOM but are not blockers — can be resolved during schematic review.

### Q11: ESP32-S3 USB Data Lines
The ESP32-S3 has native USB on GPIO19 (D−) and GPIO20 (D+). Should we:
- **A)** Route D+/D− from USB-C to ESP32-S3 for firmware flashing via USB (recommended)
- **B)** Power-only USB-C (no data, requires separate UART header for programming)

> Recommendation: **Option A** — native USB eliminates need for CP2102 and allows direct firmware flashing.

### Q12: GPS Backup Battery
The NEO-M9N supports a backup coin cell for hot start (~2s vs 24s cold start). Should we:
- **A)** Include a CR1220 holder on PCB (adds ~5mm height, ~3,000₫)
- **B)** Skip backup battery (cold start every power cycle, saves space)

### Q13: PTC Fuse on VBUS
Should we add a resettable PTC fuse (e.g., 500mA or 1A) on the USB VBUS line for overcurrent protection?
- **A)** Yes, add 1A PTC fuse (0805 package, ~1,000₫)
- **B)** No, rely on BQ24074 internal current limiting

> Recommendation: **Option A** — adds a layer of safety for 1,000₫.

### Q14: Test Points
Should we include labeled test points on the PCB for:
- 3.3V rail, VBAT, GND, SPI signals, I2C signals, UART GPS?
- Useful for debugging v2.0 prototypes

> Recommendation: **Yes** — critical for a development/audit version.
