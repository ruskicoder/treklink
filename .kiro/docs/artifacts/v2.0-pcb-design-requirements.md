## 1. PCB Design Requirements

### 1.1 Functional Requirements

**Primary Functions:**
- LoRa 433MHz mesh communication (Meshtastic protocol)
- GPS position tracking (4-constellation GNSS)
- 9-axis motion sensing (accelerometer + gyroscope + magnetometer)
- OLED display output (128×64 monochrome)
- User input via 5 buttons (SELECT, SOS, UP, DOWN, POWER)
- Audio/haptic notifications (buzzer PWM, vibration motor)
- USB-C charging with power path management (charge + operate simultaneously)
- Li-Ion battery management (1S2P 21700, 10Ah)
- Bluetooth 5.0 LE for mobile app pairing
- Native USB for firmware flashing (no external UART bridge)
- Soft-latch power management (phone-style button control: hold 3s ON, click = screen toggle, hold 3s OFF)

**Input/Output Interfaces:**

| Interface | Type | Qty | Purpose |
|-----------|------|-----|---------|
| USB-C Breakout | 4-pin plated pads | 1 | Charging (5V/1A) + USB data (D+/D−) to ESP32-S3 |
| SPI | Internal bus | 1 | MCU ↔ LoRa radio (E22-400M22S) |
| UART | Internal bus | 1 | MCU ↔ GPS module (NEO-M9N) |
| I2C | Internal bus | 1 | MCU ↔ OLED (0x3C) + IMU (0x68) |
| GPIO | Digital I/O | 5 | Buttons (SELECT, SOS, UP, DOWN, POWER) |
| PWM | Output | 1 | Passive buzzer |
| GPIO | Output (via NPN) | 1 | Vibration motor |
| GPIO | Output | 1 | Status LED |
| ADC | Analog input | 1 | Battery voltage monitoring |
| IPEX/U.FL | RF connector | 1 | 1× GPS antenna. LoRa uses built-in IPEX on E22 module. |
| Solder pads | Battery connection | 1 | 21700 battery solder pads on PCB |
| Solder pads | CR1220 backup | 1 | GPS backup battery (tabbed coin cell, hand-soldered) |

## 1.2 Performance Requirements

**MCU Clock:**
- ESP32-S3 dual-core Xtensa LX7 @ 240 MHz (internal to module)

**Signal Integrity:**

| Bus | Speed | Requirements |
|-----|-------|-------------|
| SPI (LoRa) | Up to 10 MHz | Short traces (≤ 30mm), matched lengths, ground plane beneath |
| I2C | 400 kHz (Fast Mode) | 4.7kΩ pull-ups on SDA/SCL, max bus length ~100mm |
| UART (GPS) | 9600–115200 baud | Standard routing, no special impedance |
| USB 2.0 | 12 Mbps (Full Speed) | 90Ω differential impedance, D+/D− matched length |
| RF (GPS) | 1.57 GHz | **50Ω controlled impedance** trace from NEO-M9N RF pin to IPEX receptacle. Solid GND plane beneath, ground shield vias surrounding RF path. |

**Critical RF Layout Rules:**
- LoRa antenna connects directly to the built-in IPEX receptacle on the E22-400M22S module. No external RF routing on PCB is needed for LoRa.
- 50Ω microstrip trace from NEO-M9N RF_IN pad to GPS IPEX connector. Include 0Ω placeholder for active bias-T upgrade path + ESD protection diode.
- Solid, unbroken ground plane (Layer 2) directly beneath the GPS RF trace.
- No signal traces crossing under RF traces.
- E22 module must have a solid ground plane beneath (no routing on L2/L3 under module).

### 1.4 Power Requirements

**Power Architecture:**

```
USB-C Pads ──→ [PTC 1.5A] ──→ [BQ24074] ──┬──→ [TPS63802] ──┬──→ 3.3V Rail
                                           │        ↑ (EN Pin)  └──→ [AMS1117-1.8] ──→ 1.8V (IMU VDDIO)
                                           │        │
                                           │   [Soft-Latch] ←── GPIO 9 (HOLD/SENSE)
                                           │        ↑ (Button press)
                                           └── [DW01A+8205A] ── [1S2P 21700 Battery]
```

| Parameter | Value |
|-----------|-------|
| **Input** | USB-C 5V DC (wired externally to PCB breakout pads) |
| **Battery** | 1S Li-Ion, 3.0V–4.2V (2× 21700 parallel = 10Ah) |
| **System rail** | 3.3V (via TPS63802 buck-boost) |
| **Max system current** | ~500mA peak (LoRa TX burst) |
| **Charge current** | 1.0A (programmed via 887Ω ISET resistor) |
| **USB input limit** | 500mA (programmed via 1.6kΩ ISET2 resistor) |
| **Quiescent current** | ~11µA (TPS63802 power-save mode) |
| **Shutdown target** | < 10µA total system (phone-style off state) |
| **PTC fuse** | 1.5A hold, 30V max, SMD 1210 package |

## 2. Enclosure Integration Requirements

### 2.1 Enclosure Details
- **Model:** Prebuilt vertical plastic project box from thegioiic.
- **Outer Dimensions:** 125 × 80 × 32.5 mm.
- **Inner Space:** ~119 × 74 × 26.5 mm.
- The PCB occupies the right side of the inner space. The left side is reserved for the battery cells (43.0 mm width allocation).

### 2.2 Mechanical Openings & Drilling (Client Hand-Drilled)
- All button openings, screen window, and USB-C/antenna port holes will be hand-drilled/cut by the client using a power drill and stepped bit.
- To accommodate manufacturing tolerances and drill inaccuracies, the design uses a **completely modular button and port strategy**:
  - **No rigid ports or switches on the PCB edges** except for RF antenna receptacles.
  - USB-C port is a panel-mount cable wired back to the PCB.
  - Buttons are panel-mount waterproof buttons wired back to the PCB breakouts.
  - SMD buttons on-board are for testing and troubleshooting only.

## 4. Notes for the Designer

1. **Castellated modules are critical.** The ESP32-S3, E22-400M22S, and NEO-M9N must use castellated pad footprints. Select castellated assembly when setting up JLCPCB order options.
2. **Soft-Latch circuit implementation.** The EN pin of TPS63802 must have a 100kΩ pull-down. The power button connects VBAT to the EN pin via a Schottky diode (BAT54S). To latch the system ON, implement a dual-MOSFET latch: ESP32-S3 GPIO 9 (active HIGH latch output) drives the gate of an N-Channel MOSFET (2N7002, with a 10kΩ series resistor and 100kΩ pull-down to GND). The N-MOSFET drain pulls down the gate of a P-Channel MOSFET (AO3401A, with a 100kΩ pull-up to VBAT). The P-MOSFET drain connects to the EN pin to latch VBAT. For sensing the power button press, the designer must route the button node (isolated from VBAT via diode or resistor divider) to a separate GPIO (e.g. GPIO 13, active LOW with internal pull-up) so the MCU can detect short/long button presses safely without exposure to VBAT.
3. **USB-C breakout pads.** Provide four 2.54mm pitch plated through-holes at the bottom edge of the PCB labeled VBUS, D+, D−, GND. Keep CC1 and CC2 5.1kΩ pull-downs on-board. Place ESD diodes (USBLC6) and VBUS TVS (SMAJ5.0A) / PTC fuse (1.5A) close to the breakout pads.
4. **Direct OLED solder pads.** Provide 4-pin plated through-holes at the top center of the board. The 0.96" SSD1306 OLED module will be laid flat on the PCB and soldered directly to these pads.
5. **Button breakouts and SMD test buttons.** Provide five 2-pin plated through-hole pads for wiring external buttons (UP, SELECT, DOWN, SOS, POWER). Place small SMD tactile buttons (e.g. 3×3mm or 3×4mm) in parallel with these breakouts for bench testing.
6. **GPS active bias-T option.** Place a 0Ω resistor footprint between VCC_RF and the RF_IN microstrip path. This serves as a placeholder for upgrading to an active antenna in future revisions.
7. **CR1220 battery footprint.** Place solder pads (not a holder) on the top side of the board for a tabbed CR1220 battery. Protect the NEO-M9N V_BCKP pin with a Schottky diode (BAT54S) to prevent back-charging.
8. **ICM-20948 level shifting.** Since the ICM-20948 has VDDIO limited to 1.95V max, use an AMS1117-1.8 regulator to supply its VDDIO rail. Implement bidirectional logic level shifters (using 2N7002 N-MOSFETs) for SDA, SCL, and INT lines to translate between the 1.8V logic on the IMU side and the 3.3V logic on the ESP32-S3 side.
