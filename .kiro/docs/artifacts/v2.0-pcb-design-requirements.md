## 1. PCB Design Requirements

### 1.1 Functional Requirements

**Primary Functions:**
- LoRa 433MHz mesh communication (Meshtastic protocol)
- GPS position tracking (4-constellation GNSS)
- 9-axis motion sensing (accelerometer + gyroscope + magnetometer)
- OLED display output (128×64 monochrome)
- User input via 4 tactile buttons (MENU, SOS, UP, DOWN)
- Audio/haptic notifications (buzzer PWM, vibration motor)
- USB-C charging with power path management (charge + operate simultaneously)
- Li-Ion battery management (1S2P 21700, 10Ah)
- Bluetooth 5.0 LE for mobile app pairing
- Native USB for firmware flashing (no external UART bridge)

**Input/Output Interfaces:**

| Interface | Type | Qty | Purpose |
|-----------|------|-----|---------|
| USB-C | SMD connector (16-pin) | 1 | Charging (5V/1A) + USB data (D+/D−) to ESP32-S3 |
| SPI | Internal bus | 1 | MCU ↔ LoRa radio (E22-400M22S) |
| UART | Internal bus | 1 | MCU ↔ GPS module (NEO-M9N) |
| I2C | Internal bus | 1 | MCU ↔ OLED (0x3C) + IMU (0x68) |
| GPIO | Digital I/O | 4 | Tactile buttons (MENU, SOS, UP, DOWN) |
| PWM | Output | 1 | Passive buzzer |
| GPIO | Output (via NPN) | 1 | Vibration motor |
| GPIO | Output | 1 | Status LED |
| ADC | Analog input | 1 | Battery voltage monitoring |
| IPEX/U.FL | RF connector | 1 | LoRa antenna (→ SMA pigtail → external antenna) |
| Solder pads | Battery connection | 1 | 21700 battery holder wires |
| Solder pads | CR1220 backup | 1 | GPS backup battery (tabbed coin cell, hand-soldered) |

### 1.2 Performance Requirements

**MCU Clock:**
- ESP32-S3 dual-core Xtensa LX7 @ 240 MHz
- No special clock routing required (integrated within module)

**Signal Integrity:**

| Bus | Speed | Requirements |
|-----|-------|-------------|
| SPI (LoRa) | Up to 10 MHz | Short traces (≤ 30mm), matched lengths, ground plane beneath |
| I2C | 400 kHz (Fast Mode) | 4.7kΩ pull-ups on SDA/SCL, max bus length ~100mm |
| UART (GPS) | 9600–115200 baud | Standard routing, no special impedance |
| USB 2.0 | 12 Mbps (Full Speed) | 90Ω differential impedance, D+/D− matched length ±0.15mm |
| RF (LoRa) | 433 MHz | **50Ω controlled impedance** trace from E22 IPEX pad to edge IPEX connector. Solid ground plane beneath. Keep-out zone around RF trace. |

**Critical RF Layout Rules:**
- 50Ω microstrip trace from E22-400M22S antenna pad to IPEX connector
- Solid, unbroken ground plane (Layer 2) directly beneath RF trace
- No signal traces crossing under RF trace
- Keep-out zone: 5mm clearance around antenna/IPEX area
- E22 module must have solid ground plane beneath (no routing on L2/L3 under module)

**Critical GPS Layout Rules:**
- NEO-M9N ceramic patch antenna area must have NO copper on ANY layer beneath/around it
- Follow u-blox integration manual for antenna keep-out dimensions
- GPS module placed at PCB edge, antenna facing upward (toward top shell)
- Ground vias surrounding GPS RF traces for shielding

### 1.4 Power Requirements

**Power Architecture:**

```
USB-C 5V ──→ [PTC 1.5A] ──→ [BQ24074] ──┬──→ [TPS63802] ──→ 3.3V Rail
                                          │        ↑
                                          └── [DW01A+FS8205A] ── [1S2P 21700 Battery]
                                               (protection)
```

| Parameter | Value |
|-----------|-------|
| **Input** | USB-C 5V DC (via BQ24074 power path) |
| **Battery** | 1S Li-Ion, 3.0V–4.2V (2× 21700 parallel = 10Ah) |
| **System rail** | 3.3V (via TPS63802 buck-boost) |
| **Max system current** | ~500mA peak (LoRa TX burst) |
| **Charge current** | 1.0A (programmed via 887Ω ISET resistor) |
| **USB input limit** | 500mA (programmed via 1.6kΩ ISET2 resistor) |
| **Quiescent current** | ~11µA (TPS63802 power-save mode) |
| **Deep sleep target** | < 50µA total system |
| **PTC fuse** | 1.5A hold, 30V max, SMD 1210 package (PPTC resettable) |

**Power Consumption Estimates:**

| Component | Active | Sleep/Idle |
|-----------|--------|------------|
| ESP32-S3 | ~240mA (WiFi TX) | ~10µA (deep sleep) |
| E22-400M22S (SX1262) | ~120mA (TX +22dBm) / ~5mA (RX) | ~1µA (sleep) |
| NEO-M9N GPS | ~25mA (tracking) | ~15µA (backup) |
| ICM-20948 | ~1.2mA | ~8µA (sleep) |
| SSD1306 OLED | ~20mA | ~0µA (I2C off) |
| Total (active) | ~410mA | — |
| Total (deep sleep) | — | ~40µA |

## 6. Notes for Designer

1. **Castellated modules are critical.** The ESP32-S3, E22-400M22S, and NEO-M9N must use castellated pad footprints. Verify JLCPCB supports castellated assembly for each (select "Castellated Holes" in order form).

2. **RF trace impedance.** The 433 MHz LoRa trace from E22 IPEX pad to edge IPEX connector must be 50Ω controlled impedance microstrip. This is the single most important layout constraint.

3. **Ground plane integrity.** Layer 2 (GND) must be solid/unbroken under all RF components (E22, IPEX connector, SMA path). Power section can use Layer 3.

4. **GPS antenna keep-out.** No copper on any layer under/around NEO-M9N ceramic patch antenna. Follow u-blox integration manual dimensions.

5. **ESP32-S3 antenna keep-out.** No copper under the ESP32-S3 module's PCB antenna area. Follow Espressif hardware design guidelines.

6. **Thermal pad.** BQ24074 QFN thermal pad must connect to ground plane via thermal vias (minimum 4× 0.3mm vias under pad) for heat dissipation during 1A charging.

7. **CR1220 footprint.** Provide solder pads for CR1220 tabbed coin cell (not a holder). Client will hand-solder. Pads should be on top side of PCB near NEO-M9N, with a Schottky diode (BAT54S or similar) to prevent back-charging from 3.3V rail.

8. **PTC fuse.** Use SMD PPTC resettable fuse, 1.5A hold current, 30V max, **1210 or 1812 package** (NOT through-hole RUEF series). Place between USB-C VBUS and BQ24074 IN pin.

9. **Buttons are through-hole.** 12×12mm waterproof tactile buttons with 4 through-hole pins. Place button footprints so caps protrude through top shell.

10. **Test points.** Include all test pads listed in §1.8. These are critical for v2.0 prototype debugging.
